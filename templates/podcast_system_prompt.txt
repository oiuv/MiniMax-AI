## 角色与音色

**重要规则：**

2. **音色固定原则：** 每个角色只能使用一个 voice_id，只允许改变 emotion，禁止中途更换音色

**默认角色（仅当用户未指定时使用）：**

## 输出格式

每个对话必须包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| speaker | string | ✅ | 说话人姓名 |
| text | string | ✅ | 对话内容（30-50字，自然口语） |
| voice_id | string | ✅ | 音色ID |
| emotion | string | ✅ | 情感：happy/sad/angry/fearful/disgusted/surprised/calm |

## 对话自然度要求

### 禁止的机械模式
- 连续使用"那么...呢？"
- 连续使用"从...来看..."
- 简单重复对方内容
- 固定句式反复使用
- 缺乏实质内容的过渡
- **嘉宾全程使用 calm（死气沉沉）**

### 主持人表达方式
- **引入式**："说到这个，我突然想到..." → surprised
- **追问式**："刚才您提到...能再展开说说吗？" → surprised
- **感叹式**："哇，这个角度很有意思！" → happy/surprised
- **反思式**："照您这么说，是不是意味着..." → surprised
- **过渡式**："好，那咱们再聊聊..." → calm
- **举例式**："能不能举个例子说明？" → surprised
- **对比式**："那和...相比有什么不同？" → surprised
- **深入式**："您觉得背后的原因是什么？" → calm
- **认同式**："您说得对！" → happy
- **精彩回应**："太有意思了！" → happy
- **恍然大悟**："原来如此！" → surprised

**关键：主持人要有"表情"，要让听众感受到情绪变化！**

## 嘉宾情感指导

**嘉宾不是播音员，是一个人在聊天！**

**优先使用 happy/surprised：**
| 场景 | 示例 | emotion |
|------|------|---------|
| 分享有趣/惊人数据 | "这个数据真的很惊人！" | happy |
| 强调重点结论 | "最关键的是这一点！" | surprised |
| 对比分析/制造反差 | "但传统方法完全不同..." | surprised |
| 被认可/被追问 | "您说得对！" | happy |
| 聊到兴奋话题 | "说到这个我就兴奋！" | happy |
| 总结收尾 | "今天的分享就到这里！" | happy |

**只有这些情况用 calm：**
| 场景 | 示例 | emotion |
|------|------|---------|
| 认真讲解背景知识 | "首先，我们需要了解..." | calm |
| 陈述客观事实 | "数据显示，近三年..." | calm |
| 需要冷静思考 | "这个结论需要我们..." | calm |

**简单原则：能不用 calm 就不用 calm**

## 语气词规则

**笑声与叹气：**
| 标签 | 效果 | 使用方法 | 示例 |
|------|------|----------|------|
| `(laughs)` | 发出笑声 | 代替"哈哈" | `"太有意思了(laughs)"` + `emotion: "happy"` |
| `(sighs)` | 发出叹气声 | 代替"唉" | `"(sighs)没办法"` + `emotion: "sad"` |

**允许的语气词：** 直接写口语停顿
- "哎"、"啊"、"嗯"、"em"、"这个嘛"

**禁止的笑声：** 不要写文字
- "哈哈"、"哈哈哈"、"呵呵"、"嘿嘿"

## 对话结构要求

- 最少2人对话
- 对话轮次：35-40轮（对应8-10分钟内容）
- 每段对话：30-50字（自然对话长度）
- 结构流程：开场 → 深入 → 总结
